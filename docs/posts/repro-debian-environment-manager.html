Introducing Repro: Your Ultimate Debian Environment Manager

A declarative environment management tool for Debian systems that makes reproducing your exact development environment across machines simple.

The Problem: Reproducible Linux Environments

As Linux enthusiasts and developers, we've all faced the frustration of setting up a new machine or recreating our development environment. Remembering all the packages, tools, and configuration settings we've accumulated over time is nearly impossible. Traditional backup solutions don't understand package managers, and manual documentation is tedious and error-prone.

The Solution: Repro to the Rescue

Repro solves this by automatically tracking your package installations and creating a declarative configuration file. It's like package.json for your entire system, but designed for Debian/Ubuntu package management.

Key Features:
  • ✅ Declarative configuration - define desired state, not steps
  • ✅ Automatic tracking - knows which packages you installed
  • ✅ Multi-source support - apt, snap, flatpak, cargo, npm, pip...
  • ✅ Version pinning - ensure exact package versions
  • ✅ One-command restore - set up new machine in minutes

Installation

Install Repro

bash
# Clone the repository
git clone https://github.com/stefan-hacks/repro.git
cd repro

# Install dependencies
sudo apt install python3 python3-pip

# Install Repro
sudo pip3 install -e .

# Or use the install script
./install.sh

Quick Start

1. Initialize Your Configuration

bash
# Initialize Repro in your home directory
repro init

# This creates ~/.config/repro/packages.toml

2. Current State Detection

bash
# Scan existing packages
repro scan

# This identifies all installed packages and their sources

3. Configuration File Structure

~/.config/repro/packages.toml
# Repro configuration - manages system packages declaratively

[apt]
essential = ["git", "curl", "wget", "vim", "htop", "tmux"]
development = ["build-essential", "python3-dev", "nodejs", "npm"]
security = ["ufw", "fail2ban", "clamav"]

[apt.optional]
gui = ["firefox", "chromium", "vlc"]

[snap]
apps = ["code", "slack", "discord"]

[flatpak]
graphics = ["gimp", "inkscape"]

[scripts]
pre-install = """
#!/bin/bash
echo "Running pre-install hooks..."
sudo apt update
"""

post-install = """
#!/bin/bash
echo "Running post-install hooks..."
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
"""

Common Workflows

Add a Package

bash
# Add package to configuration and install
repro add apt htop
repro add snap code --classic
repro add flatpak org.gimp.GIMP

# Or install manually and track
sudo apt install vim
repro track vim  # Adds to config

Apply Configuration

bash
# Install all packages from config
repro apply

# Dry run to see what would change
repro apply --dry-run

# Apply specific category only
repro apply --category development

Sync State

bash
# Sync config with current system state
repro sync

# Removes packages from config that are no longer installed
# Adds newly installed packages to config

Version Control Integration

Since your configuration is a simple TOML file, version control it like any other code:

bash
# Create a dotfiles repo
git init ~/dotfiles
cd ~/dotfiles

# Link repro config
ln -s ~/.config/repro/packages.toml .

# Commit your environment
git add packages.toml
git commit -m "Initial environment configuration"

# Push to GitHub
git push origin main

Setting Up a New Machine

With Repro, setting up a new Debian machine takes minutes instead of hours:

bash
# On new machine:
sudo apt update && sudo apt install git

# Clone your dotfiles
git clone https://github.com/yourusername/dotfiles.git

# Install Repro
cd dotfiles
./install-repro.sh

# Apply your complete environment
repro apply

# Done! Your environment is identical to your previous machine
