AppArmor Cheat Sheet for Linux System Administrators

Master Mandatory Access Control with profiles, enforcement modes, and security hardening techniques for Ubuntu and Debian systems.

Introduction

AppArmor is a Mandatory Access Control (MAC) system for Linux that restricts program capabilities through profiles. Unlike SELinux which uses labels, AppArmor works with file paths — making it simpler to configure while providing robust security.

1. Check AppArmor Status

Quickly verify if AppArmor is running and see loaded profiles:

bash
sudo apparmor_status
# OR
sudo aa-status

2. View AppArmor Profiles

Profiles are stored in /etc/apparmor.d/ with a specific naming convention:

bash
# List all profiles
ls /etc/apparmor.d/

# View disabled profiles
ls /etc/apparmor.d/disable/

# Example: profile for /usr/bin/man
# Located at: /etc/apparmor.d/usr.bin.man

3. Enable/Disable Profiles

First, install the utilities:

bash
sudo apt update && sudo apt install apparmor-utils

Enable a Profile (Enforce Mode)

bash
sudo aa-enforce /etc/apparmor.d/usr.sbin.rsyslogd

# Alternative: enforce by binary path
sudo aa-enforce /usr/sbin/httpd

Disable a Profile (Complain Mode)

bash
sudo aa-complain /etc/apparmor.d/usr.sbin.rsyslogd

4. Creating Custom Profiles

Use the interactive profiler to capture application behavior:

bash
sudo aa-genprof /usr/bin/scp

# In another terminal, run the application:
scp ~/test remote_user@remote_IP:~/

# Back in profiler:
# Press 'S' to scan logs
# Choose (A)llow or (D)eny for each action
# Save with 'S' then 'F' to finish

5. Modify Profile from Logs

Adjust existing profiles based on audit logs:

bash
sudo aa-logprof

6. View AppArmor Logs

Monitor DENIED events and policy violations:

bash
sudo apt install apparmor-notify
sudo aa-notify -s 1 -v

Understanding the Modes

Mode Behavior Use Case
Enforce Policy enforced, violations blocked and logged Production environments
Complain Violations logged only, not blocked Testing new profiles

Service Management

bash
sudo systemctl stop apparmor
sudo systemctl restart apparmor
sudo systemctl status apparmor
Tip: AppArmor profiles load automatically at boot. Changes to profiles require a service restart to take effect.
