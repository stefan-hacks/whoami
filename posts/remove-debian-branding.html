Remove Debian Branding: Eliminate the Blue Plymouth Boot Screen

February 2026 5 min read

The blue Plymouth splash screen between GRUB and GDM comes from Debian's default branding. Here's how to remove it for a clean, professional boot experience with LUKS encryption support intact.

Why Remove Plymouth?

  • Cleaner boot sequence
  • No graphical artifacts
  • Custom GRUB themes visible
  • Faster boot perception

Step 1: Remove Plymouth

sudo apt purge plymouth plymouth-themes
sudo apt autoremove

Step 2: Update GRUB Configuration

Edit /etc/default/grub and remove splash from the default command line:

# Change this:
GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"

# To this (completely silent):
GRUB_CMDLINE_LINUX_DEFAULT="quiet"

# Or this (show boot messages):
GRUB_CMDLINE_LINUX_DEFAULT=""

Step 3: Disable Debian Theme Script

The blue background is set by /etc/grub.d/05_debian_theme. Remove execute permissions to skip it:

sudo chmod -x /etc/grub.d/05_debian_theme

Note: This is safer than deleting the file — package updates won't recreate it, and you can reverse the change if needed.

Step 4: Apply Changes

sudo update-grub
sudo update-initramfs -u -k all
sudo reboot

Result

Your boot sequence will now be:

  1. Custom GRUB menu
  2. Black screen (or kernel messages)
  3. LUKS password prompt (if encrypted)
  4. GDM login screen

No more blue Debian splash screen! Clean and professional.

Verification

Confirm the Debian theme script is skipped:

grep -A2 "05_debian_theme" /boot/grub/grub.cfg

This should return empty — no cyan/blue color lines in the output.

Reversing the Change

If you want the blue screen back:

sudo chmod +x /etc/grub.d/05_debian_theme
sudo update-grub
sudo update-initramfs -u -k all
